# VYBE / Pomoika

Telegram Mini App для знакомств и общения внутри Telegram. Приложение запускается через бота, авторизует пользователя по Telegram-данным и открывает мобильный интерфейс с анкетами, лентой, входящими событиями и настройками профиля.

Проект сделан как клиент-серверное приложение:
- **frontend** отвечает за интерфейс Mini App;
- **backend** обрабатывает авторизацию, бизнес-логику, работу с анкетами, лайками, комментариями и уведомлениями;
- **PostgreSQL** хранит пользователей, анкеты, фотографии, реакции, жалобы и служебные данные.

## О приложении

Это сервис знакомств в формате Telegram Mini App с низким порогом входа. Пользователю не нужно проходить отдельную регистрацию, подтверждать почту или скачивать отдельное мобильное приложение. Достаточно открыть Web App из Telegram, после чего можно сразу перейти к заполнению анкеты и просмотру других пользователей.

Основная идея приложения — быстрые знакомства в компактном формате. Пользователь создает анкету, добавляет фотографии, указывает базовые параметры и после этого получает доступ к ленте. В ленте можно просматривать анкеты, ставить лайки, оставлять комментарии, пропускать неподходящие профили и реагировать на входящие действия.

При этом приложение не ограничивается только механикой "лайкнул и ушел". Внутри есть дополнительные сценарии взаимодействия: ответы на комментарии, повторные реакции, подарки, ежедневные бонусы, реферальная логика, мини-опросы и уведомления о новых событиях. За счет этого продукт ближе не только к dating-mini app, но и к легкой социальной платформе внутри Telegram.

## Что есть в приложении

### 1. Профиль пользователя
После входа пользователь попадает в профиль — это основной экран, откуда начинается работа с приложением.

В профиле можно:
- посмотреть данные Telegram-аккаунта;
- увидеть статус своего аккаунта;
- создать или отредактировать анкету;
- управлять фотографиями;
- включать или отключать отдельные режимы;
- открыть настройки;
- посмотреть доступные лайки и связанные ограничения;
- перейти к дополнительным функциям, если для пользователя доступна роль администратора.

Профиль выступает как точка управления всем аккаунтом: здесь хранится основная информация о пользователе, состоянии его анкеты и доступных действиях.

### 2. Анкета
Анкета — это центральная сущность приложения. Именно она показывается в ленте другим пользователям и именно на ее основе строятся основные сценарии знакомства.

В анкете можно указать:
- имя или ник;
- возраст;
- город;
- пол;
- параметры видимости;
- описание о себе;
- фотографии.

Пользователь может не просто один раз создать анкету, а редактировать ее по мере необходимости: обновлять текст, менять фото, корректировать параметры отображения и подстраивать профиль под нужный формат показа в ленте.

### 3. Лента анкет
Лента — это главный пользовательский сценарий приложения. В ней отображаются анкеты других людей, с которыми можно взаимодействовать.

Внутри ленты пользователь может:
- просматривать карточки других людей;
- открывать анкету подробнее;
- ставить лайки;
- писать комментарии;
- пропускать анкеты;
- использовать фильтры;
- переключаться между обычным просмотром и быстрым сценарием взаимодействия.

Лента ориентирована на быстрые действия. Пользователь не тратит много времени на сложную навигацию: основной сценарий построен вокруг коротких решений и последовательного просмотра профилей.

### 4. Лайки и реакции
В приложении есть система лайков как отдельная внутренняя механика. Лайк — это не просто иконка реакции, а важное действие, которое влияет на дальнейшее взаимодействие между пользователями.

Поддерживаются такие сценарии:
- обычный лайк анкеты;
- ответный лайк;
- повторные реакции;
- анонимные действия в отдельных сценариях;
- ограничения по количеству лайков;
- восстановление доступных лайков по времени.

Из-за этого пользовательский поток становится более управляемым: реакции не полностью бесконечны, а встроены в продуктовую механику удержания и вовлечения.

### 5. Комментарии и ответы
Кроме лайков в приложении предусмотрены комментарии. Это делает знакомство менее случайным и дает возможность отреагировать на анкету более содержательно.

Пользователь может:
- оставить комментарий к анкете;
- получить входящий комментарий;
- ответить на комментарий;
- удалить или скрыть часть взаимодействий в рамках интерфейса сообщений.

За счет комментариев приложение поддерживает более живой формат общения, чем классический свайп-сервис, где все ограничивается только взаимными лайками.

### 6. Сообщения и входящие события
В проекте есть отдельный раздел, где собираются все входящие действия, связанные с анкетой пользователя.

Туда попадают:
- новые лайки;
- комментарии;
- ответы на комментарии;
- повторные реакции;
- отдельные служебные события, связанные с активностью вокруг анкеты.

Этот раздел нужен не как полноценный мессенджер, а как центр входящих событий. Пользователь видит, кто проявил активность, и может быстро перейти к ответному действию: открыть анкету, поставить лайк в ответ, посмотреть детали или продолжить общение уже через Telegram.

### 7. Взаимные симпатии и переход к общению
Приложение ориентировано не только на просмотр ленты, но и на доведение пользователя до следующего шага — реального контакта внутри Telegram.

Когда между пользователями возникает взаимный интерес, сценарий не обрывается на экране ленты. Приложение помогает довести взаимодействие до личного общения: показать входящее событие, дать ответить и упростить переход к диалогу.

Именно поэтому Mini App выступает как оболочка для знакомства, а сам Telegram — как естественная среда для дальнейшего общения.

### 8. Ежедневный бонус и streak-механика
В проекте есть механика ежедневного бонуса. Она нужна для возврата пользователя в приложение и поддержания активности.

Пользователь может:
- получать ежедневную награду;
- накапливать последовательность посещений;
- возвращать ресурсы, нужные для активности внутри сервиса;
- поддерживать прогресс через streak-логику.

Такие элементы усиливают удержание и добавляют ритм использования: приложение мотивирует открывать его не один раз, а регулярно.

### 9. Подарки и внутренняя активность
Помимо базовых лайков, в проекте предусмотрены дополнительные способы взаимодействия. Один из них — подарки и специальные действия, связанные с внутренними ресурсами приложения.

Это делает поведение пользователя более разнообразным:
- можно не только лайкать;
- можно выделять интерес дополнительным действием;
- события в ленте и сообщениях становятся более насыщенными;
- интерфейс получает больше сценариев вовлечения.

### 10. Мини-опросы и дополнительные сигналы интереса
В проекте есть механика мини-опросов, встроенных в ленту. Они нужны не как отдельный большой раздел, а как дополнительный способ собрать быстрые интересы пользователя.

Такие сигналы используются для того, чтобы:
- сделать взаимодействия менее случайными;
- добавить контекст к реакции;
- повысить качество совпадений;
- разнообразить обычный сценарий просмотра анкет.

По смыслу это легкая игровая и социальная механика, которая расширяет знакомство за пределы стандартного лайка.

### 11. Уведомления
Приложение может уведомлять пользователя о новых действиях. Это нужно, чтобы не терять входящие реакции и возвращать человека к приложению.

Уведомления касаются, например:
- новых лайков;
- комментариев;
- ответов;
- других событий активности.

Часть событий видна внутри интерфейса Mini App, а часть может быть связана с Telegram-уведомлениями через бота.

### 12. Настройки
В приложении есть раздел настроек, где пользователь может управлять отдельными параметрами поведения приложения.

В зависимости от текущей реализации там доступны сценарии вроде:
- управления уведомлениями;
- информации о приложении;
- переключения темы;
- переключения языка;
- удаления аккаунта или связанных действий с учетной записью.

Это делает приложение не просто набором экранов, а полноценным управляемым сервисом с собственной пользовательской конфигурацией.

### 13. Безопасность, жалобы и модерация
Для продукта такого типа особенно важна модерация, и в проекте это учтено.

В приложении есть:
- обязательное принятие пользовательских условий;
- отправка жалоб на анкеты;
- ограничения для нарушителей;
- бан и временная блокировка;
- административные инструменты;
- проверки пользовательского контента.

Отдельно предусмотрена работа с изображениями: загрузка фото проходит обработку, а также есть логика, связанная с проверкой неподходящего контента. Это важно для поддержания качества анкет и общей безопасности среды.

### 14. Админские возможности
В проекте есть административная часть, которая нужна для ручного контроля пользовательской базы и спорных случаев.

Администратор может работать с:
- пользователями;
- жалобами;
- ограничениями и блокировками;
- отдельными параметрами оформления или событийной логики.

Эта часть не является основной для обычного пользователя, но важна для реальной эксплуатации приложения.

## Как работает приложение

Типичный пользовательский сценарий выглядит так:

1. Пользователь открывает Mini App из Telegram-бота.
2. Приложение получает данные Telegram Web App и авторизует пользователя на backend.
3. Если пользователь новый, для него создается запись в базе.
4. Пользователь принимает условия использования.
5. Затем он заполняет анкету: добавляет фото, возраст, город и описание.
6. После этого становится доступна лента анкет.
7. В ленте пользователь смотрит другие профили, ставит лайки, пишет комментарии или пропускает анкеты.
8. Если кто-то взаимодействует с его анкетой, это появляется в разделе сообщений и событий.
9. При взаимной симпатии приложение подводит пользователя к дальнейшему общению уже внутри Telegram.

Таким образом, приложение охватывает полный цикл: от входа и создания профиля до первых реакций, совпадений и перехода в реальное общение.

## Основные разделы интерфейса

По структуре проекта в приложении выделяются такие крупные части:

- **Profile** — профиль пользователя, анкета, настройки состояния аккаунта;
- **Feed** — лента анкет и основной сценарий просмотра;
- **Messages** — входящие лайки, комментарии, ответы и другие события;
- **Settings** — настройки приложения и предпочтений;
- **Daily Bonus** — ежедневные награды и streak-механика;
- **Terms Modal** — принятие пользовательских условий;
- **Anketa / View / Comment Modal** — модальные окна для просмотра анкеты и быстрых действий;
- **Admin Panel** — административные инструменты для модерации.

## Запуск локально

### 1. Установить зависимости

```bash
cd frontend
npm install

cd ../backend
npm install
```

### 2. Настроить переменные окружения

Для backend нужен `.env`. Минимально используются такие переменные:

```env
DATABASE_URL=
BOT_TOKEN=
JWT_SECRET=
PORT=4000
```

Для frontend обычно указывается адрес backend API:

```env
VITE_API_URL=
```

### 3. Поднять базу данных

Нужно запустить PostgreSQL локально или через Docker.

### 4. Подготовить Prisma

```bash
cd backend
npx prisma generate
npx prisma db push
```

### 5. Запустить приложение

Backend:

```bash
cd backend
npm run dev
```

Frontend:

```bash
cd frontend
npm run dev
```

## Запуск через Docker Compose

```bash
docker compose up --build
```

Обычно таким способом поднимаются:
- frontend;
- backend;
- PostgreSQL;
- дополнительные сервисы, если они предусмотрены конфигурацией.
