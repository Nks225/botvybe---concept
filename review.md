5. Проект
5.1. Описание проекта

Запуск из Telegram.
Создание анкеты.
Просмотр ленты.
Взаимодействие с анкетами (лайк, комментарий, пропуск).
Переход к диалогу в Telegram при взаимной симпатии.
Проект ориентирован на mobile-first сценарий, минимальный порог входа и высокую скорость первого целевого действия.

5.2. Цели проекта
Уменьшить трение на входе за счет авторизации через Telegram.
Обеспечить понятный и быстрый UX внутри 3 основных вкладок: Профиль, Лента, Сообщения.
Сформировать замкнутый цикл вовлечения: просмотр анкеты → действие → входящее событие → ответ.
Обеспечить базовую безопасность среды: модерация, жалобы, бан, скрытие/удаление нарушителей.
Сделать техническую базу для масштабирования: realtime, модульная архитектура, dockerized запуск.
5.3. Границы проекта (scope)
В рамках проекта реализуются:

Telegram-авторизация через initData.
Анкета пользователя с фото, возрастом, городом, полом, описанием.
Лента анкет с фильтрацией и свайп/кнопочными действиями.
Лайки, комментарии, ответы на комментарии.
Взаимные лайки и переход к общению в Telegram.
Realtime обновления через SSE.
Настройки уведомлений.
Удаление аккаунта и восстановление в ограниченное окно.
Система жалоб.
Админ-панель модерации.
Вне базового scope (можно в развитие):

Продвинутый ML-ранжировщик.
Платежные сценарии и монетизация.
Полноценный встроенный чат внутри Mini App.
Георадиус и геокарта.
5.4. Роли в системе
User — создает анкету, взаимодействует с лентой, получает сообщения, отправляет жалобы.
Admin — обрабатывает жалобы, банит/разбанивает, удаляет аккаунты нарушителей, просматривает анкеты для модерации.
5.5. Функциональные модули проекта
Модуль A. Авторизация и доступ
Вход пользователя выполняется через Telegram WebApp.
Проверяется валидность initData.
Проверяется статус пользователя: активен, заблокирован, удален, не принял условия.
При необходимости показываются ограничивающие экраны: Terms, Ban, Restore.
Модуль B. Профиль и анкета
Пользователь создает или редактирует анкету.
Поля анкеты: город, возраст, пол, видимость в фильтрах, описание, фото.
Редактируется отображаемое имя (ник).
Поддерживается анонимный режим профиля.
Доступно удаление анкеты.
Модуль C. Лента
Лента формируется на основе релевантности.
Пользователь видит карточку, фото-карусель, базовую информацию, описание.
Действия: Like, Comment, Skip.
Есть фильтрация по полу.
Есть сценарии пустой ленты.
Есть форма жалобы на текущую анкету.
Модуль D. Сообщения
В один поток объединяются лайки, комментарии, ответы и матчи.
Пользователь может удалять входящие элементы.
На комментарий можно отправить ответ.
На входящий лайк можно ответить лайком или пропустить.
При взаимном лайке появляется CTA на переход в Telegram-диалог.
Модуль E. Realtime
Клиент подписывается на SSE-канал.
Получает события о новых лайках/сообщениях/обновлениях.
Обновляет интерфейс без перезагрузки.
Поддерживается heartbeat и реконнект.
Модуль F. Модерация
Пользователь отправляет жалобу (типовая причина + optional текст).
Жалобы поступают в админ-интерфейс.
Админ принимает решение: отклонить, бан, удаление.
Система поддерживает автоматические ограничительные меры при массовых жалобах.
Модуль G. Аккаунт и настройки
Тогглы уведомлений (лайки/сообщения).
Удаление аккаунта пользователем.
Восстановление в ограниченное время.
Принудительный logout при бане/удалении.
Модуль H. Лимиты и реферальная механика
Пользователь имеет лимит лайков.
Лимит восстанавливается по таймеру.
За приглашение друга начисляется бонус лайков.
При исчерпании лайков выводится сценарий приглашения.
Модуль I. Мини-опрос
В ленте показывается карточка с выбором интереса.
Ответ хранится ограниченное время.
При совпадении ответов у пары можно усиливать match-сигнал.
5.6. Ключевые бизнес-правила проекта
Минимальный возраст пользователя анкеты: 14.
Возраст в анкете: 14–99.
Фото в анкете: 1–3.
Максимальная длина комментария/ответа/сообщения жалобы: 256.
Лимит лайков на пользователя: 100.
Интервал восстановления лайков: 3 часа.
Бонус за реферала: +50 лайков.
Окно восстановления удаленного аккаунта: 24 часа.
Cooldown повторной жалобы на того же пользователя: 2 часа.
Автобан при экстремальном количестве жалоб: временное ограничение.
Анонимный режим скрывает идентифицирующие данные в ряде сценариев.
5.7. Техническая архитектура проекта
Frontend: React + TypeScript + Vite + styled-components.
Backend: Node.js + Express.
Database: PostgreSQL.
ORM: Prisma.
Media processing: multer + sharp.
Realtime: SSE.
Infra: Docker Compose (frontend, backend, db).
Архитектурный принцип:

Тонкий клиент с фокусом на UX и состояниях.
Backend содержит бизнес-правила и контроль доступа.
База данных хранит доменную модель пользователей, анкет, событий, жалоб и модерации.
5.8. Интеграции
Telegram WebApp API:

Получение initData.

Управление поведением webview.

Открытие ссылок/чатов Telegram.

Telegram Bot API:

Отправка push-сообщений пользователю при офлайн-состоянии.

Ретеншн-уведомления для возврата в приложение.

5.9. Этапы реализации проекта
Этап 1. Базовый каркас.
Авторизация, навигация, структура БД, базовый UI.
Этап 2. Анкета и медиа.
Формы, загрузка фото, валидации, сохранение.
Этап 3. Лента и действия.
Подбор анкет, свайпы, лайк/комментарий/пропуск.
Этап 4. Сообщения и матчи.
Поток событий, ответы, сценарий взаимного лайка.
Этап 5. Realtime и уведомления.
SSE, обновления счетчиков, офлайн-нотификации.
Этап 6. Модерация и админка.
Жалобы, бан/разбан/удаление, force logout.
Этап 7. Hardening.
Оптимизация, тесты, документация, релизная стабилизация.
5.10. Что является результатом проекта (deliverables)
Рабочее Telegram Mini App (app) с полным пользовательским циклом.
Backend API с реализованными правилами домена.
Миграции БД и схема данных.
Реализованный realtime-канал.
Админ-панель модерации.
Документация по запуску и эксплуатации.
5.11. Критерии готовности (Definition of Done)
Пользователь может войти через Telegram и создать анкету без блокирующих ошибок.
Лента отдает релевантные анкеты и корректно обрабатывает действия.
Сообщения обновляются в realtime и корректно отображаются.
Взаимный лайк формирует рабочий сценарий перехода в Telegram-общение.
Жалоба пользователя доступна и обрабатывается админом.
Бан/удаление немедленно ограничивают доступ пользователя.
Система стабильно запускается локально и в docker-окружении.
Ключевые пользовательские сценарии проходят smoke-тест.
Документация актуальна фактическому поведению системы.
